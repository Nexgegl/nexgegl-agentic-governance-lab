# Board Response Skill

## 1. Purpose
يصوغ رداً موجزاً على مستوى مجلس إدارة (Board-Level) لسؤال أو استفسار، خالياً من الحشو والمبالغة، بنسخة عربية و/أو إنجليزية حسب الطلب.

## 2. When to Use
- عند الحاجة لصياغة رد رسمي لعضو مجلس إدارة أو مستثمر على سؤال مباشر.
- عند تلخيص موقف لعرضه في اجتماع مجلس إدارة.
- **لا يُستخدم** لصياغة قرارات مالية أو استراتيجية فعلية — فقط لصياغة **الرد النصي** على استفسار حولها.

## 3. Inputs Required

**Required:**
- السؤال أو الاستفسار الأصلي من عضو المجلس.
- الحقائق المتاحة للإجابة (بيانات، حالة مشروع، رقم مالي).

**Optional:**
- اللغة المطلوبة (عربي فقط / إنجليزي فقط / كلاهما).
- إن كان الرد يحتاج الإشارة لخطر أو قرار معلَّق.

**Missing Input Handling:**
إن لم تتوفر إجابة مؤكدة لجزء من السؤال، يُذكر ذلك صراحة في الرد ("سيُتابَع ويُوافى المجلس لاحقاً") بدل صياغة رد مطمئن بلا أساس.

## 4. Operating Rules
- Do not invent facts. لا يُذكر رقم أو التزام لم يُؤكَّد في المدخلات.
- افصل بين ما هو مؤكَّد وما هو "قيد المتابعة" في الرد نفسه، دون خلطهما بصياغة توحي باليقين.
- **No fluff. No overclaiming.** كل جملة يجب أن تحمل معلومة أو موقفاً، لا عبارات إنشائية عامة.
- **لا قرار مؤسسي يُصاغ كأنه نافذ** ضمن الرد ما لم يكن بالفعل معتمداً عبر Evidence + Authority + Audit — إن كان الرد يحمل زاوية قرار معلَّق، يُذكر أنه "قيد الاعتماد" لا كأمر واقع.
- **Signal is not Decision** — أي مؤشر أداء يُذكر في الرد يُصاغ كمعطى، لا كنتيجة نهائية قاطعة ما لم تكن كذلك فعلاً.

## 5. Execution Workflow
1. حدد السؤال الجوهري الذي يطرحه عضو المجلس فعلياً (لا الأسئلة الفرعية الهامشية).
2. اجمع الحقائق المؤكدة ذات الصلة المباشرة فقط.
3. حدد إن كان الرد يحمل زاوية خطر (Risk) أو قرار معلَّق يستحق الذكر.
4. صِغ الرد بأقصر صياغة ممكنة دون فقدان الدقة.
5. أنتج نسخة عربية و/أو إنجليزية حسب الطلب — كل نسخة مستقلة بذاتها وليست ترجمة حرفية ركيكة للأخرى.
6. راجع الرد النهائي: هل يحتوي أي جملة حشو يمكن حذفها دون فقدان معنى؟ احذفها.

## 6. Output Format

```
BOARD RESPONSE — [الموضوع/السؤال المختصر] — [التاريخ]

السؤال الأصلي: [نص السؤال أو ملخصه]

النسخة العربية:
[رد مباشر، جملتان إلى أربع جمل كحد أقصى، بلا حشو]

English Version (إن طُلب):
[Direct reply, two to four sentences max, no fluff]

زاوية الخطر/القرار (إن انطبقت):
[جملة واحدة تحدد إن كان هناك خطر معلَّق أو قرار ينتظر اعتماداً، أو "لا يوجد" إن لم تنطبق]
```

## 7. Quality Gates

| الحالة | معيار قابل للملاحظة |
|---|---|
| **PASS** (رد جاهز للإرسال) | الرد يجيب مباشرة على السؤال الأصلي في 4 جمل أو أقل لكل نسخة لغة، لا حشو، لا ادعاء غير مدعوم بحقيقة. |
| **FIX** | الرد يجيب لكنه أطول من اللازم أو يحتوي جملة حشو واحدة يمكن حذفها، أو النسخة العربية والإنجليزية غير متسقتين في المضمون. |
| **FAIL** | الرد يحتوي ادعاءً غير مؤكَّد بصياغة قاطعة (Overclaiming)، أو يتجاهل زاوية خطر واضحة كان يجب ذكرها. |

## 8. Anti-Patterns
- افتتاح الرد بعبارات مجاملة طويلة قبل الوصول للمضمون.
- استخدام لغة تسويقية مطمئنة ("كل شيء تحت السيطرة تماماً") دون سند فعلي.
- ترجمة النسخة الإنجليزية حرفياً من العربية بأسلوب ركيك بدل صياغتها بلغة إنجليزية طبيعية لمجلس إدارة.
- إغفال ذكر خطر معروف لصانعي القرار لمجرد أنه "غير مريح".

## 9. Example Prompt
"استخدم Board Response Skill لصياغة رد على سؤال عضو مجلس الإدارة: 'لماذا تأخر إطلاق ميزة X في ESTARED شهراً كاملاً؟' بالعربية والإنجليزية معاً."

## 10. Example Output Skeleton

```
BOARD RESPONSE — [الموضوع] — [YYYY-MM-DD]
السؤال الأصلي: [نص]
النسخة العربية: [رد موجز]
English Version: [concise reply]
زاوية الخطر/القرار: [جملة أو "لا يوجد"]
```
