# Routine: تقرير انحراف الحوكمة الأسبوعي (Weekly Governance Drift Report)
## v1.0

> يُقرأ مع `00-master-standards/NEXGEGL_CLAUDE_MASTER.md` §5 (قواعد مكافحة الانحراف).

---

## 1. الغرض (Purpose)

رصد أي انحراف تراكمي (Drift) عن المعيار الرئيسي والـ Product Profiles عبر الوقت — انحراف قد لا يظهر في مراجعة PR فردية، لكنه يتراكم عبر عدة تغييرات صغيرة.

## 2. التفعيل (Trigger)

- جدولة أسبوعية ثابتة (مثال: كل يوم اثنين).
- يمكن تشغيله يدوياً عند الشك في وجود انحراف.

## 3. ما يُفحص

1. **اتساق التسمية:** هل ظهرت أي إشارة لاسم بديل لـ ESTARED/إسترد أو أي منتج آخر في الكود، التوثيق، أو رسائل Commit خلال الأسبوع؟
2. **سلامة سلسلة القرار:** هل أُدخل أي منطق جديد يُصدر فعلاً مؤسسياً دون المرور بـ Evidence/Authority/Audit؟
3. **تجاوزات مكتشفة سابقاً:** هل تكررت مشكلة سبق الإبلاغ عنها في تقرير PASS/FIX/FAIL سابق دون إصلاح فعلي؟
4. **اتساق تصنيف المستودعات:** هل تغيّر تصنيف أي مستودع (Core IP/Product/Client/Experiment) دون تحديث CLAUDE.md المحلي؟
5. **تحديث المعيار الرئيسي مقابل الواقع:** هل ظهرت ممارسات فعلية متكررة تستدعي تحديث `00-master-standards/` نفسه (لا كسره، بل تطويره رسمياً)؟

## 4. الوكلاء المشاركون

- **`crag`** يقود هذا التقرير، بالتنسيق مع أي وكيل آخر (`product-governor`, `security-rls-auditor`, إلخ) حسب ما يكتشفه من مؤشرات في نطاقه.

## 5. صيغة التقرير

```
Weekly Governance Drift Report — أسبوع [تاريخ البداية] إلى [تاريخ النهاية]

1. اتساق التسمية: [لا انحراف / انحراف مكتشف: التفاصيل]
2. سلامة سلسلة القرار: [سليم / انحراف مكتشف: التفاصيل]
3. تكرار مشاكل سابقة: [لا تكرار / قائمة المشاكل المتكررة]
4. اتساق تصنيف المستودعات: [متسق / تعارضات مكتشفة]
5. توصية تحديث المعيار الرئيسي: [لا حاجة / توصية محددة]

الخلاصة: [لا انحراف يستدعي تصعيداً / يستدعي مراجعة بشرية عاجلة]
```

## 6. التصعيد

أي انحراف مكتشف في هذا التقرير **لا يُصحَّح ضمنياً**. يُوثَّق في التقرير، ويُرفع لصاحب الصلاحية البشرية المسؤول عن حوكمة NEXGEGL لاتخاذ القرار بشأن التصحيح.
