# Routine: مراجعة ما قبل الدمج (PR Pre-Merge Review)
## v1.0

> يُقرأ مع `00-master-standards/NEXGEGL_CLAUDE_MASTER.md` و`03-sub-agents/`.

---

## 1. الغرض (Purpose)

ضمان أن أي Pull Request يُدمَج في أي مستودع NEXGEGL يمر عبر مراجعة حوكمية موحدة قبل الدمج، بحيث لا يدخل انحراف (Drift) أو خطر أمني أو مالي أو تصميمي دون رصد.

## 2. التفعيل (Trigger)

- عند فتح PR جديد.
- عند كل تحديث (push) لاحق على نفس PR.
- عند طلب مراجعة يدوية صريحة من مساهم بشري.

## 3. الوكلاء المفعّلون (Agents Invoked)

بالترتيب:

1. **`crag`** — فحص الاتساق العام ومكافحة الانحراف. (إلزامي دائماً)
2. **`product-governor`** — فحص اتساق الهوية والنطاق التجاري. (إلزامي إن كان المستودع Product/Client Repo)
3. **`security-rls-auditor`** — فحص أمني وصلاحيات. (إلزامي إن مسّ التغيير طبقة بيانات أو صلاحيات)
4. **`cfo-logic-reviewer`** — فحص منطق مالي. (إلزامي إن مسّ التغيير حسابات أو تدفقات مالية)
5. **`ux-rtl-reviewer`** — فحص واجهة وRTL. (إلزامي إن مسّ التغيير واجهة مستخدم أو نصوصاً)
6. **`qa-test-reviewer`** — فحص تغطية الاختبار. (إلزامي دائماً)

> يُحدَّد أي وكيل ينطبق فعلياً بناءً على نوع الملفات المتغيرة في الـ PR (diff scope detection)، ولا حاجة لتشغيل كل الوكلاء إن لم يكن التغيير يمس نطاقهم.

## 4. قاعدة تجميع النتيجة النهائية (Final Aggregation Rule)

- أي **FAIL** واحد من أي وكيل مفعَّل → النتيجة الإجمالية **FAIL**، ويُمنع الدمج فوراً، ويُصعَّد للمراجعة البشرية.
- لا يوجد FAIL لكن يوجد **FIX** واحد أو أكثر → النتيجة الإجمالية **FIX**، يُرجَع PR مع قائمة إصلاحات مجمَّعة، ولا يُدمج حتى تُعاد المراجعة وتصدر PASS.
- **PASS** من كل الوكلاء المفعَّلين → يُسمح بالدمج (مع بقاء أي متطلبات موافقة بشرية إضافية معمول بها في المستودع).

## 5. صيغة التقرير الموحد

```
PR Pre-Merge Review — [رقم/رابط PR]
التصنيف: [Core IP/Product/Client/Experiment]
التاريخ: [تاريخ]

النتائج الفرعية:
- CRAG: [PASS/FIX/FAIL] — [ملاحظة موجزة]
- Product Governor: [PASS/FIX/FAIL/N/A] — [ملاحظة]
- Security/RLS Auditor: [PASS/FIX/FAIL/N/A] — [ملاحظة]
- CFO Logic Reviewer: [PASS/FIX/FAIL/N/A] — [ملاحظة]
- UX/RTL Reviewer: [PASS/FIX/FAIL/N/A] — [ملاحظة]
- QA/Test Reviewer: [PASS/FIX/FAIL] — [ملاحظة]

النتيجة الإجمالية: [PASS/FIX/FAIL]
الإجراء: [يُسمح بالدمج / يُعاد للإصلاح / يُمنع الدمج ويُصعَّد]
```

## 6. ملاحظة على النطاق

هذا Routine يوثّق **آلية** المراجعة. تفعيله الفعلي (كـ GitHub Action أو Hook) يتم عند دمج هذا الإطار في مستودع منتج فعلي، ولا يُنفَّذ آلياً من هذا المستودع التجريبي وحده.
