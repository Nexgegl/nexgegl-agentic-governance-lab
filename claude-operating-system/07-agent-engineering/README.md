## Foundational Principles

### Principle 1 — Agents Do Not Decide

Agents may:

- research
- analyze
- summarize
- recommend
- explain
- validate
- escalate

Agents must not independently approve institutional decisions.

### Principle 2 — Signal Is Not Decision

A signal is not a decision.

A recommendation is not a decision.

A generated output is not a decision.

The approved flow is:

Signal
↓
Evidence
↓
Analysis
↓
Recommendation
↓
Authority
↓
Decision
↓
Execution

### Principle 3 — Governance First, Not AI First

The correct order is:

Policy
↓
Authority
↓
Evidence
↓
AI
↓
Decision

AI must operate inside governance boundaries.

Governance must not be retrofitted after automation.

### Principle 4 — AI Explains, Rules Decide

AI explains.

Rules decide.

Authority approves.

Audit records.

Execution follows approval.

### Principle 5 — Fail Closed, Never Fail Open

If authority is missing, stop.

If policy is missing, stop.

If evidence is missing, stop or escalate.

If auditability is missing, stop.

If runtime boundaries are unclear, stop.

## Mandatory Governance Layers

Any enterprise agent system must define:

### Authority Layer

Who has the right to approve or execute?

### Policy Layer

What is allowed and forbidden?

### Evidence Layer

What evidence supports the action?

### Validation Layer

Is the input, context, and output valid?

### Audit Layer

Can the action be inspected later?

### Compliance Layer

Does the action violate law, regulation, policy, or contract?

### Escalation Layer

When must the agent stop and escalate?

### Monitoring Layer

Is the agent behaving correctly?

### Cost Layer

What does the agent cost to run?

### Runtime Enforcement Layer

How are violations prevented, not merely reported?
